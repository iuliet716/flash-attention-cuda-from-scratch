# Step 6. Bank Conflict Avoidance — Swizzling (한글 초안)

> 최종 영어 문서 작성용 참고 초안.

## 1. 개념

### Shared memory의 뱅크 구조

shared memory는 4B(1워드) 단위로 **32개 뱅크**에 번갈아 배정된다.
워드 인덱스를 32로 나눈 나머지가 뱅크 번호다.

```
주소(워드):  0   1   2  ...  31  32  33  ...
뱅크:        0   1   2  ...  31   0   1  ...
```

한 사이클에 각 뱅크는 서로 다른 워드 1개만 내줄 수 있다.

- 워프의 32개 lane이 32개 **다른 뱅크** → 1사이클에 완료 (이상적)
- n개 lane이 **같은 뱅크의 다른 워드** → n사이클로 직렬화 = **n-way 뱅크 충돌**
- 여러 lane이 **같은 워드** → 브로드캐스트되므로 충돌 아님

### 우리 커널의 충돌: 최악의 패턴

QK^T 내적에서 lane i는 K의 i번째 행을 읽는다:

```cuda
Ktile[lane * d + k]   →   뱅크 = (lane * d + k) % 32
```

d가 32의 배수이므로 `lane * d % 32 == 0`. 즉 **모든 lane이 뱅크 `k % 32` 하나에
몰린다** — 32-way 충돌, 내적 루프의 모든 읽기가 32배로 직렬화된다.

step 05의 float4 읽기에서는 하드웨어가 워프를 8-lane씩 4개 페이즈로 나눠 처리하는데,
페이즈마다 8개 lane이 같은 뱅크를 때려 **8-way 충돌이 남아 있다**. step 05가 4배
개선하고도 여전히 charge가 남은 이유다.

### 해결책 두 계열

1. **패딩**: 행 스트라이드를 d+1로 늘려 행마다 뱅크가 어긋나게. 간단하지만
   float4 정렬(16B)이 깨지고 공간이 낭비된다.
2. **XOR 스위즐**: 저장할 때 행마다 청크 순서를 재배열. 공간 그대로, 정렬 유지.
   → 채택.

### XOR 스위즐 공식

```
물리 청크 위치 = 논리 청크 번호 ^ (행 번호 % 8)
```

XOR의 성질상, 연속된 8개 행은 같은 논리 청크를 서로 다른 8개 슬롯에 저장하게 된다.
그래서 float4 접근의 한 페이즈(8개 lane = 8개 행)가 8개의 서로 다른 청크 슬롯
= 32개 뱅크 전부를 정확히 한 번씩 커버한다 → conflict-free.

```
      논리:            물리(스위즐 후):
행0:  0 1 2 3 ...      0 1 2 3 ...      (^0)
행1:  0 1 2 3 ...      1 0 3 2 ...      (^1)
행2:  0 1 2 3 ...      2 3 0 1 ...      (^2)
행3:  0 1 2 3 ...      3 2 1 0 ...      (^3)
```

읽을 때도 같은 공식을 쓰면 되므로 **추가 메모리도, 별도의 역변환도 없다**.
XOR는 "8청크 그룹 내부의 순열"이므로 행의 청크 수(d/4)가 8의 배수여야 인덱스가
범위를 벗어나지 않는다 → `d % 32 == 0` 제약.

### 왜 K만 스위즐하나

충돌은 접근 패턴이 만든다. 세 타일의 패턴이 다르다:

- **Q**: `Qtile[warp * d + k]` — 워프 안 모든 lane이 같은 주소 → 브로드캐스트, 충돌 없음
- **V**: `Vtile[c * d + lane + 32*i]` — lane이 연속 워드 → 뱅크 0~31 고르게, 충돌 없음
- **K**: lane이 행 인덱스 → 위에서 본 32-way 충돌

그래서 K 타일만 스위즐한다. "충돌이 있는 곳에만 처방"이 코드도 설명도 단순하게 만든다.

## 2. 코드 구현 설명

### 스위즐 함수

```cuda
__device__ __forceinline__ int swizzle(int r, int c4) {
    return c4 ^ (r % 8);
}
```

### 저장 시 (K만 스위즐, V는 그대로)

```cuda
Ktile4[rl * d4 + swizzle(rl, c4)] = in ? Kb4[...] : zero4;
Vtile4[idx] = in ? Vb4[...] : zero4;
```

스레드마다 (행, 청크) 쌍이 유일하고 XOR는 행 안에서 전단사(1:1)이므로
쓰기끼리 겹칠 일도 없다.

### 읽기 시 — 같은 공식

```cuda
const float4* k4 = reinterpret_cast<const float4*>(Ktile) + lane * d4;
...
const float4 b = k4[swizzle(lane, k)];  // conflict-free
```

### 호스트 코드

`TORCH_CHECK(d % 32 == 0, ...)` — 스위즐이 8청크 그룹 안에서만 섞이기 위한 조건.
